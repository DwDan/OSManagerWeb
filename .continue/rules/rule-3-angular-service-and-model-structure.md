You must follow the service and model organization rules of this Angular project.

## Services

- Services must be created under `src/app/services/[entity]/`
- Always group service files by entity
- Do NOT place services inside component folders
- Create `.spec.ts` files when requested

## Models / Interfaces / Responses

- Models must be created under `src/app/models/[entity]/`
- Use subfolders such as `responses` and `requests` when requested
- Do NOT place models inside component folders
- Do NOT place models inside service folders

## Critical behavior

- If the user provides exact file paths, those paths are mandatory and must not be changed
