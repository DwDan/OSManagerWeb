Your rule contentYou must follow the UI pattern of the project, which uses PO UI.

## UI Framework

- Use PO UI components whenever possible
- Do NOT use plain HTML elements if PO UI equivalent exists

## Layout

- Use PO Page components when applicable (po-page-default or similar)
- Maintain spacing and layout consistency with existing components

## Tables / Lists

- Prefer PO Table for list screens
- Use consistent column definitions
- Keep labels in Portuguese

## Forms

- Use PO UI form components (inputs, switches, etc.)
- Keep forms simple and consistent with existing screens

## UX

- Maintain consistency with existing screens
- Do NOT invent new layouts or design patterns
- Follow existing components as reference

## Behavior

- Keep UI simple and functional
- Avoid overengineering
